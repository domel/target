  • Images (in DICOM and JPEG)
  • BM*_CT directories: Multimodal CT in the context of acute stroke assessment refers to the combination of a non-contrast CT (NCCT) scan of the brain, a CT angiogram of the extracranial and intracranial neck vessels, and a CT perfusion scan of the entire brain.
  • BM*_RM directories: The sequences of a brain MRI.
  • BM*.jpg files: An electrocardiogram.
  • inside:
  • Patient ID: represents the individual patient ID.
  • Study ID: refers to the study. For hospital del Mar data mean basically whether images belong to a baseline or a follow-up study.
  • Series ID: this corresponds to the acquisition ID. For instance, whether the image is a FLAIR, T1, T2...so, it's basically the type of the image...(https://dicom.innolitics.com/ciods/cr-image/general-series/0020000e)
  • dcm: dicom files. Each files represents an image, generally (but not always) obtained at the axial plane. Each dicom file contains a dicom header with the metadata.

Isansys

  • Vital Signs. The minute by minute data shown on the gateway and in the patient charts which are accessible on the Isansys Lifeguard Dashboard. Note the entry 61441 is an error message indicating that no HR measurement can be made. These usually occur at the start of the session when the Lifetouch is being fitted, or if the device disconnects as you saw. If Temperature, Blood Pressure, weight etc were part of the session data it would be shown here.
  • Session info. Connectivity status, device activity and device log information for the session.
  • Gateway audit. log A record of all interactions by a user with the Gateway
  • LT Heart Beats. This is the fundamental heartbeat data we use to calculate the Heart Rate and Respiration Rate. The columns show the relative RS amplitude and the temporal position of the R peak as shown by the Timestamp in milliseconds (the ECG is sampled at 1000 samples per second). For convenience (e.g. for HRV studies) we calculate the R-R interval and also display an associated simple 3 level activity measure if this feature is activated. Note that the RS amplitudes are not scaled in mV, but are in fact ADC counts, in a direct linear relationship to mV.
  • Lifetouch Set Up Mode Logs. A log showing when the Set Up (ECG) mode was switched on. The actual 100Hz Set-Up Mode (full ECG) data is in the separate file “qwerty234__Created_26Jan2024@1705_LT_Setup_Mode” and the Set Up Mode Logs tell you where to look for each of the ECG recordings you made. You can reconstruct the ECG from these values using a simple Line Chart function in Excel
  • Lifetouch Raw Accelerometer. Values for each of the x, y and z outputs of the accelerometer at 10Hz sampling frequency.
  • Pulse Ox Set Up Mode Logs. Similar to the Lifetouch Set Up Mode Logs, but for the PPG signal from the pulse oximeter. The actual 75 Hz Set-Up Mode (full PPG) data is in the separate file “qwerty234__Created_26Jan2024@1705_PO_Setup_Mode” and the Set Up Mode Logs tell you where to look for each of the PPG recordings you made. You can reconstruct the PPG from these values using a simple Line Chart function in Excel
  • Pulse Ox dd-mm-yyyy. These are the oxygen saturations measurements uploaded every 1 seconds. The minute-by-minute values shown on the gateway are calculated from these.
